# DevTools — Windows Dev Process Manager

> Потому что `taskkill /F /IM node.exe` надоедает набирать в 23:00

---

Набор PowerShell-скриптов, которые делают ровно то, о чём ты думаешь каждый раз, когда VS Code отказывается запускать сервер: **убивают всё лишнее, чистят память, показывают что происходит и пишут бортовой журнал сессии**.

---

## Что внутри

### `killdevs.ps1` — убийца процессов

Находит всё живое: Node, Python, Go, PHP, Java, .NET и компанию. Показывает таблицу — кто сколько жрёт CPU и RAM, на каком порту висит. Дальше ты сам решаешь: прибить всех сразу или выбрать конкретных виновников.

Умеет ещё:
- чистить оперативку (Standby List + Working Set) — когда браузер с IDE поели всё до крошки
- мониторить RAM с уведомлениями в трее, чтобы не удивляться фризам
- замечать незнакомые процессы на портах — мало ли кто там живёт
- добавлять свои процессы в список через меню
- ставить автозапуск при выключении ПК через Task Scheduler, чтобы к следующему сеансу всё было чисто
- показывать все открытые порты системы одним нажатием

### `porttimemachine.ps1` — машина времени портов

Пишет историю каждого порта: когда открылся, когда закрылся, сколько прожил. История сохраняется в JSON между сессиями — через месяц можно открыть и посмотреть что запускалось в конкретный день. Четыре режима: живой список текущих портов, полная история, поиск по порту или процессу, статистика — топ портов, топ процессов, активность по дням в виде графика.

### `envguardian.ps1` — охранник .env файлов

Сканирует все папки проектов и находит каждый `.env` файл. Для каждого проверяет две вещи: покрыт ли он `.gitignore` (если нет — красный DANGER) и содержит ли реальные секреты — пароли, API ключи, токены, JWT, AWS ключи, Stripe live ключи, URLs с паролем в строке (WARN). Следит за изменениями в реальном времени — как только `.env` меняется, сразу пересканирует и кидает уведомление в трей. Значения в отчёте маскируются — видно только первые 4 символа.

### `crashcoroner.ps1` — детектив по падениям процессов

Висит в фоне и ждёт. Как только dev-процесс неожиданно умирает — за секунды собирает всё что можно: последние строки лога, RAM и CPU в момент смерти, список свидетелей (что ещё работало рядом), записи Windows Event Log. Пишет дело в файл и кидает уведомление в трей. Жил меньше 30 секунд — CRITICAL, не успел запуститься. Жил дольше — WARNING или INFO. Незаменимо когда сервер падает и не понятно почему.

### `devblackbox.ps1` — бортовой журнал сессии

Запускаешь в начале рабочего дня — и он молча пишет всё что происходит: какие процессы стартовали и когда упали, сколько прожили, какие порты занимали, пики RAM, ошибки из логов. Когда нажмёшь Q — генерирует текстовый отчёт с полной хронологией. Через неделю можно открыть и вспомнить что вообще делал в среду.

### `devlogs.ps1` — просмотрщик логов

Сканирует порты, находит активные серверы и тянет их логи в реальном времени. Уровни подсвечены (`ERROR` красным, `WARN` жёлтым, остальное — как глаза требуют). HTTP-статусы тоже: `2xx` — зелёный, `4xx` — оранжевый, `5xx` — ты знаешь что делать.

Плюс:
- поиск сразу по всем лог-файлам — не надо помнить, где что лежит
- сохранение дампа с датой в имени файла
- настройка папок поиска под свою структуру проекта

## Установка

**Шаг 1.** Забери скрипты:

```powershell
git clone https://github.com/ВАШ_НИК/devtools.git C:\Scripts
```

Или просто скопируй файлы в `C:\Scripts\` — работает так же.

**Шаг 2.** Разреши PowerShell их запускать (один раз, не страшно):

```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```

**Шаг 3.** Создай ярлыки на рабочем столе, чтобы не печатать каждый раз:

*DEV KILLER* — правой кнопкой на рабочем столе → Создать → Ярлык:
```
powershell -Command "Start-Process powershell '-NoExit -File C:\Scripts\killdevs.ps1' -Verb RunAs"
```

*DEV LOGS:*
```
powershell -Command "Start-Process powershell '-NoExit -File C:\Scripts\devlogs.ps1' -Verb RunAs"
```

## Чтобы devlogs видел твои серверы

Логи нужно писать в файл. Проще всего — через `Tee-Object`, это не больно:

**Node.js** — в `package.json`:
```json
"scripts": {
  "dev": "node server.js 2>&1 | Tee-Object -FilePath app.log",
  "start": "node server.js 2>&1 | Tee-Object -FilePath app.log"
}
```

**Python:**
```powershell
python app.py 2>&1 | Tee-Object -FilePath app.log
uvicorn main:app --reload 2>&1 | Tee-Object -FilePath app.log
```

**Go:**
```powershell
go run main.go 2>&1 | Tee-Object -FilePath app.log
```

**PHP:**
```powershell
php -S localhost:8080 2>&1 | Tee-Object -FilePath app.log
```

## Конфигурация

| Файл | Для чего |
|------|----------|
| `killdevs-config.ps1` | Свои процессы поверх стандартных |
| `devlogs-config.ps1` | Свои папки для поиска логов |
| `devlogs-saved/` | Сохранённые дампы логов |
| `devblackbox-sessions/` | Отчёты по завершённым сессиям |
| `crashcoroner-cases/` | Дела по упавшим процессам |
| `envguardian-report.txt` | Последний отчёт по .env файлам |
| `porttimemachine-history.json` | История всех портовых сессий |

Все эти файлы в `.gitignore` — там локальные пути, в репозиторий им не место.

## Права администратора

Часть функций работает только с правами админа:

- убийство системных процессов
- очистка Standby List
- автозапуск через Task Scheduler

Запускай ярлыки через «Запуск от имени администратора» — и всё будет доступно.
